If using external HAProxy
-------------------------
• Point your HAProxy backend servers at the **node IPs** on NodePort {{ $nodePort }} (not the pod DNS names).
• Ensure "send-proxy-v2" (PROXY v2) is enabled towards VerneMQ.
• Keep the internal listener for cluster-local clients and health probes.

Example (simplified):
  backend vernemq_nodes
    mode tcp
    balance roundrobin
    option tcp-check
    server cp1 <node1-ip>:{{ $nodePort }} send-proxy-v2 check
    server cp2 <node2-ip>:{{ $nodePort }} send-proxy-v2 check
    server cp3 <node3-ip>:{{ $nodePort }} send-proxy-v2 check

Notes
-----
• All tunables are sourced from values.yaml; no baked-in constants in templates.
• For message backlog while a downstream subscriber is down:
  - Publish enriched messages with QoS=1 (or 2 if you're brave) via .Values.enrich.qos.
  - Ensure the downstream subscriber uses a persistent session and subscribes with QoS 1,
    so VerneMQ will spool while it's offline.
